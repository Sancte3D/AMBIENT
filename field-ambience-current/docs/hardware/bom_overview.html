For the PCB engineer (Aron) — start here

✅ The schematic is already drawn and split into 8 pages. Open kicad/field_ambience.kicad_pro in KiCad 9 — the 8 sheets (listed below) load with every symbol, net, value and a pre-assigned footprint. You do not redraw the schematic. Then: ERC → Update PCB from Schematic (F8) → place → route.

The schematic is generated by kicad/generate_kicad_project.py — that script is the source of truth. Don't hand-edit the .kicad_sch; change the generator and re-run. There is no .kicad_pcb yet — the board layout is greenfield, that's your job. ERC/DRC run in the KiCad 9 GUI (no kicad-cli here).

4-layer stack (ADR-0018, locked)
  • L1 signals+parts · L2 solid GND (never cut) · L3 power (+3V3/+5V) · L4 signals
  • USB-D± together, equal length, over solid GND, ~90 Ω — verify on JLC's calculator
  • Class-D + boost switching away from analog audio; one ground plane, no split
  • Crystal <3 mm to MCU + local GND; amp bulk cap <5 mm from PVDD
Place these first (enclosure-fixed)
  • USB-C · both 3.5 mm jacks · display header J3
  • 5 cell switches (19 mm grid) · 5 modifier buttons · 4 encoders
  • speaker headers · battery JST · then power, MCU+decoupling, audio, I/O

✅ The 5 cells are digital, but keep a REAL keyswitch feel — read this before placing them (r18.75, ADR-0013 superseded). The cells changed from Gateron-magnetic + Hall-sensor (analog velocity) to a digital on/off switch — but NOT the same small tactile button as the modifiers (that was r18.73, and it made the playable cells feel identical to a plain modifier button, killing the "these are keyboard keys" UX). r18.74 first tried a Kailh Choc hot-swap socket, but that socket had no clean manufacturer part number and needed fiddly small-SMD hand-soldering technique that isn't obvious if you haven't built a hobbyist keyboard before. Fixed in r18.75: the Kailh Choc V1 switch (CPG135001D01) is now directly soldered to the board — 2 THT legs through 2 holes, soldered from the back, exactly the same technique as every other button/connector in this design. No hot-swap socket, no special technique, but the switch is now permanent (not swappable without desoldering). Footprint + 3D STEP pulled straight from LCSC/EasyEDA for this exact part (field_ambience:SW_KailhChoc_CPG1350_THT_2P) — the same method this repo already uses for its other custom footprints (TS-1088, MST-12D18, PJ-320D), not a third-party keyboard library. Electrically unchanged since r18.73: one pin → MCP23017 GPIO (GPA0–GPA4, nets CELL1..5_BTN), other pin → GND; MCP internal pull-up (idle=HIGH, pressed=LOW), one shared INT line — still the HiChord Batch 4+ topology (switch → I²C GPIO-expander → MCU), still no Hall sensor, no RC filter, no STM32 ADC pin used (PC0/PC1/PA4/PB0/PB1 stay freed). Caps are 3D-printed and clip onto the switch's stem. Trade-off unchanged: you lose analog press-depth/velocity — fine for triggering; Hall would only be needed for genuine expressive depth (see ADR-0013).

Assembly — who solders what. JLC reflow-assembles every SMD part (the "JLC" rows in the BOM). The hand-place rows (THT headers + modifier buttons + the 5 cell keyswitches, encoders) and the off-board rows (display module, speakers, battery, knobs) are fitted by hand, after JLC — by whoever does final assembly. So the board comes back from JLC fully SMD-populated; the THT parts + modules + mechanical parts are then hand-fitted/soldered.

Display = a finished module (this changes how it's attached). The Waveshare 1.9″ LCD is its own PCB (ST7789 + level-shifter + backlight) — it is NOT reflow-soldered to the main board. Put on the main board only J3, a 1×8 2.54 mm header: 1 VCC · 2 GND · 3 SCK (PA5) · 4 MOSI (PA7) · 5 RES (PC5) · 6 DC (PC4) · 7 CS (PA6) · 8 BLK (backlight via Q2 / PCA9685 ch15). The module is then joined by hand two ways: (a) socket on J3 + header on the module → plug-in, nothing on the module to solder, serviceable; or (b) a short 8-wire cable J3↔module → lets the display sit behind the bezel window with the PCB below (most mechanical freedom). ⚠ The module's pin order varies by vendor — verify the real module vs J3 before fixing it. Is this normal / are there better displays? Yes, module-on-header is the standard, reliable choice at this scale (same as Adafruit/Waveshare-based products). The more-integrated alternative — a bare ST7789 panel + an FPC connector soldered to the main board (lower profile) — is a much bigger job (FPC connector + panel sourcing/mounting, and you'd re-add the level-shifter + backlight the module already has); not worth it here.

Thermal: no ventilation slots (~1.5–2.2 W). Give the LDO copper + thermal vias; keep the LiPo away from LDO/charger/boost.

Workflow → fab: regenerate → open field_ambience.kicad_pro in KiCad 9 → ERC (0 err) → Update PCB from Schematic (F8) → place → route → DRC → export Gerber + CPL; BOM = kicad/jlc_bom.csv → upload to JLC.

Power-off block (ADR-0016) — already wired in the generated sheets since r18.81 (nothing to add). In power_tree.kicad_sch: U_PWR TPS22918 (C131941) gates the LDO input — VIN←+5V_RAIL, VOUT→+5V_SW→LDO(U5) VIN, ON←PWR_ON (QOD→VOUT active discharge, CT floats — per TI SLVSD76C); SW_PWR slide switch (r18.85: ALPS SSSS811101, C109335) common(2)→PWR_ON, throw(1)→+5V_RAIL; R_PWR_PD 100 kΩ PWR_ON→GND (default OFF); C_PWR_SW 10 µF on +5V_SW. Result: the side switch is a real off, while the charger sits ahead of it (still charges when off). Your job: ERC it in the GUI + buzz the switch terminals on the physical part before fab (fail-safe if wrong).

Hard blockers before order: ERC pass · layout + routing · PCB outline + mechanical coords (TBD) · USB impedance check · the power-off block above.

  • ✅ KiCad-standard: everything in standard packages — LQFP-100, SOT-23/89, SOIC, TSSOP, SSOP, 0603/0805/1210, pin headers, HRO USB-C.
  • ⚠ Modifier buttons (5× HX B3F, C36498965, THT): verify the THT footprint SW_TC1212-7.3_THT_4P fits the 12×12 4-pin THT body of the HX B3F; if the pinout differs, generate a fresh one (easyeda2kicad --full --lcsc_id=C36498965).
