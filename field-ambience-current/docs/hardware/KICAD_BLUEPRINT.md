# KiCad Blueprint ‚Äî Field Ambience (idiot-proof, step by step)

**Stand: r18.40 (2026-06-21).** This is the practical "what do I click, where,
in which order" guide. It assumes you have never laid out a PCB. Read top to
bottom once, then use it as a checklist.

> **THE ONE THING YOU MUST UNDERSTAND FIRST**
>
> **You do NOT draw the schematic.** It is generated by a Python script
> (`kicad/generate_kicad_project.py`). All ~80 components, every wire, every
> net name, and every footprint assignment already exist. You never place a
> resistor or draw a wire by hand in the schematic editor.
>
> Your real job is only two things:
> 1. **Open the project and run ERC** (electrical rule check) ‚Äî confirm the
>    generated schematic is clean.
> 2. **Do the PCB layout** ‚Äî there is no `.kicad_pcb` yet. This is the actual
>    hand-work: place footprints, route copper, run DRC, export Gerbers.
>
> So when you wonder "where do I connect what" ‚Äî it's already connected. This
> doc shows you how to *read/verify* the connections (so you trust them) and
> then how to *physically lay them out*.

---

## 0. Install + open (5 minutes)

1. Install **KiCad 9** (kicad.org/download). Not 7, not 8 ‚Äî the generator
   writes the KiCad-9 file format (`KICAD_VERSION_TAG = "20231120"`).
2. (Only if you ever change parts) regenerate the schematic:
   ```bash
   cd field-ambience-current/kicad
   python3 generate_kicad_project.py
   ```
   You normally DON'T need this ‚Äî the `.kicad_sch` files are already committed.
3. Open **`field-ambience-current/kicad/field_ambience.kicad_pro`** in KiCad
   (File ‚Üí Open Project, pick the `.kicad_pro`).
4. The Project window shows the schematic (`.kicad_sch`) and an empty PCB slot.

## 1. First-time library setup (do once)

The project uses three kinds of footprints. Two come bundled in the repo and
are already wired via `kicad/fp-lib-table`; the rest are KiCad's standard
libraries that ship with KiCad 9.

| Library | Where | Already configured? |
|---|---|---|
| KiCad standard (Package_QFP, Resistor_SMD, ‚Ä¶) | ships with KiCad 9 | ‚úÖ via your global fp-lib-table |
| `field_ambience` (6 custom footprints) | `kicad/libraries/field_ambience.pretty/` | ‚úÖ via project `fp-lib-table` (`${KIPRJMOD}` path) |
| Keyswitch (Kailh/stabilizer) | `kicad/libraries/keyswitch-kicad-library/` | ‚úÖ via project `fp-lib-table` |

**What to click once:**
- **Preferences ‚Üí Manage Symbol Libraries** ‚Üí tab "Project Specific Libraries"
  ‚Üí confirm nothing is red. If KiCad asks to migrate libraries, accept.
- **Preferences ‚Üí Manage Footprint Libraries** ‚Üí tab "Project Specific
  Libraries" ‚Üí you should see `field_ambience` (actively used everywhere,
  incl. the cell keyswitch SW1‚Äì5 as of r18.75), plus `Switch_Keyboard_Hotswap_Kailh`

  resolving ‚Äî reopen via the `.kicad_pro`, not the `.kicad_sch`.

## 2. Schematic phase ‚Äî what to DO and what NOT to do

### DO NOT
- ‚ùå draw wires
- ‚ùå place components
- ‚ùå change resistor values in the GUI (change them in the generator + re-run)
- ‚ùå "fix" anything by editing the `.kicad_sch` directly ‚Äî the next generator
   run overwrites it. The generator is the source of truth.

### DO
1. Open the root schematic. You'll see **8 hierarchical sheets** (double-click
   a sheet box to go in, breadcrumb at top to come back):
   `power_tree`, `stm32h743`, `lcd`, `mcp`, `encoder`, `audio`, `battery`.
2. **Tools ‚Üí Update Symbols from Library** (refreshes R/C/L/LED/GND symbols to
   your installed KiCad-9 versions ‚Äî harmless, recommended).
3. **Inspect ‚Üí Electrical Rules Checker ‚Üí Run ERC.**
4. Compare the result against
   [`ERC_DRC_CHECKLIST.md`](ERC_DRC_CHECKLIST.md):
   - **0 errors** required.
   - Only the *intended* warnings listed there are allowed (power-pin-not-driven
     on the +5V bridge labels, PB3/SWO not connected, single MIDI_TX label,
     MCP reserve pins). Anything else must be understood before layout.
5. If ERC is clean per the checklist ‚Üí schematic phase is DONE. Move to ¬ß4.

> Why bother if it's generated? Because the generator's structural checks
> (paren balance, 100/100 pin connectivity, hier‚Üîroot crossref) are NOT the
> same as KiCad's electrical check. The GUI ERC is the gate before layout
> (`PCB_LAYOUT_STATUS.md` blocker **B3**). It has never been run in a GUI yet ‚Äî
> you are the first to do it.

## 5. Schematic ‚Üí PCB (the transfer ‚Äî this CREATES the empty board)

1. From the schematic editor: **Tools ‚Üí Open PCB Editor** (or from the project
   window, double-click the `.kicad_pcb` slot ‚Äî KiCad offers to create it).
2. In the PCB editor: **Tools ‚Üí Update PCB from Schematic** (hotkey **F8**).
3. Dialog appears ‚Äî leave defaults, click **Update PCB**. Now ALL footprints
   appear in a pile off to the side, with thin "ratsnest" lines showing every
   connection from the schematic. **This is your wiring made physical** ‚Äî you
   never drew it, the netlist carries it over.
4. Click **Close**. You now have an unrouted board.

> Re-run F8 any time the schematic changes ‚Äî it syncs new parts/nets without
> destroying your placement.

## 6. Board setup (do this BEFORE placing/routing) ‚Äî Tools ‚Üí Board Setup

### 6.1 Layer stack-up (SPEC ¬ß9)
- **File ‚Üí Board Setup ‚Üí Board Stackup ‚Üí Physical Stackup**: set **4 layers**.
- Assign: **F.Cu = Signal**, **In1.Cu = GND** (solid pour), **In2.Cu = +5V**
  (power), **B.Cu = Signal**. Board thickness **1.6 mm**.

### 6.2 Design Rules (Board Setup ‚Üí Design Rules ‚Üí Constraints)
JLCPCB 4-layer econ capabilities:
- Min track width: **0.15 mm** (set clearance default 0.15 mm too)
- Min via: **0.3 mm drill / 0.45 mm diameter** (annular ‚â• 0.075 mm)
- Min hole-to-hole: 0.5 mm

### 6.3 Net classes (Board Setup ‚Üí Net Classes) ‚Äî set track widths
| Class | Track width | Apply to |
|---|---|---|
| Default (signal) | 0.25 mm | everything |
| Power | 0.5 mm | +3V3 |
| HighCurrent | 0.8 mm | +5V, VBAT, boost SW node, speaker outputs |
| GND | use the pour, not tracks | ‚Äî |

(The generator's `.kicad_pro` already sets `copper_line_width = 0.2` as the
editor default; the net classes above override per-net where current needs it.)

## 7. Placement + routing order (the actual layout work)

Place in this order ‚Äî group by sub-circuit, keep loops short:

1. **Fixed/mechanical first** (these are constrained by the enclosure ‚Äî see
   `PCB_FOOTPRINT_RISK_AUDIT.md ¬ß6` + `mechanical/coordinates/`):
   USB-C (board edge), audio jack (board edge), LCD header, the 5 cell switches
   (SW1‚Äì5) on the 19 mm grid, 5 modifier buttons (SW6‚Äì10), 4
   encoders, battery connector, mounting holes.
2. **Power section**: USB-C ‚Üí F1 ‚Üí TPS61089 + L1 + bulk caps. Keep the boost
   loop (U8 ‚Üî L1 ‚Üî C_OUT) as tight as possible ‚Äî this is the #1 power rule
   (SPEC ¬ß5.1, TI datasheet ¬ß10). Then the LDO.
3. **MCU + decoupling**: STM32 in the middle, each VDD's 4.7 ¬µF+100 nF pair
   *right next to its pin*. Crystal Y1 + its 2 caps as close to PH0/PH1 as
   possible with a guard.
4. **Audio**: PCM5102A close to the MCU's SAI pins, its decoupling < 5 mm,
   single-star GND between audio and digital, ferrite beads in the speaker
   lines (SPEC ¬ß5.1 audio-layout-constraints ‚Äî read those 5 points).
5. **I/O**: MCP23017, PCA9685, LEDs near their buttons/cells.

**Then route:**
- **Pour GND on In1.Cu** (Place ‚Üí Add Filled Zone ‚Üí pick GND net, draw over
  whole board). Pour **+5V on In2.Cu**. Right-click a zone ‚Üí **Fill All Zones**
  (hotkey **B**) after routing.
- Route power (thick) first, then clocks/I¬≤S, then the rest. Use the net-class
  widths from ¬ß6.3.
- Right-click a track ‚Üí **Properties** to change width; right-click a pad ‚Üí
  **Highlight Net** to see where it goes.

## 8. Verify + export (the order-ready gate)

1. **Inspect ‚Üí Design Rules Checker ‚Üí Run DRC.** Target per
   `ERC_DRC_CHECKLIST.md`: **0 errors, 0 clearance/mechanical/routing
   warnings.** Unconnected nets = forgotten routing.
2. **3D view (Alt+3)**: check nothing collides, parts sit where the enclosure
   expects. Compare against `mechanical/3d_models/MANIFEST.md` heights.
3. **1:1 paper print** of the board outline + cutouts, hold the real
   USB-C/jack/encoder/cells against it (audit ¬ß11).
4. **Export Gerbers**: File ‚Üí Fabrication Outputs ‚Üí Gerbers + Drill files.
5. **BOM + CPL** for JLC assembly: File ‚Üí Fabrication Outputs ‚Üí BOM, and the
   Placement (CPL) file. Cross-check LCSC numbers against `BOM_MASTER.md`.
6. In JLCPCB's web uploader: load Gerbers, then BOM+CPL, then **walk the
   assembly preview part by part** ‚Äî look for flipped pin-1 markers and any
   "footprint not found". Pay attention to the 6 custom footprints.

## 9. Cheat-sheet ‚Äî right-click / hotkeys you'll actually use

**Schematic editor:**
- `E` ‚Äî edit properties of selected item
- Inspect ‚Üí Electrical Rules Checker ‚Äî the ERC
